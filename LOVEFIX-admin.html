LOVEFIX 後台

連接你的 GitHub 倉庫

這個後台會把每一位客戶的資料與成品網頁，直接存進你自己的 GitHub 帳號，並用 GitHub Pages 把成品網頁上線。

請注意：免費方案下 GitHub Pages 必須使用「公開（public）」倉庫，代表每位客戶的照片、情書文字、音樂檔案，理論上任何人只要拿到確切的網址或倉庫連結都能看到（不會被主動列在搜尋結果或首頁，但並非真正加密私密）。若客戶內容較敏感，建議提醒客戶妥善保管專屬連結，不要公開分享。
⚠️ 如果你是直接雙擊打開這個檔案（網址列開頭是 file://），下面的「連接並初始化」會失敗，出現「Failed to fetch」——這不是壞掉，是瀏覽器本身不允許本機檔案直接連上外部網站，任何工具都無法繞過。請改用以下手動方式做「僅限第一次」的設定：
  1. 到 github.com/new 建立一個新倉庫（Public，勾選 Add a README file）
  2. 進倉庫 Settings → Pages → Source 選 Deploy from a branch → Branch 選 main、資料夾選 / (root) → Save
  3. 回到倉庫首頁 → Add file → Upload files → 把這個 LOVEFIX-admin.html 拖進去 → Commit changes
  4. 等 30～60 秒，改用網址打開：https://你的帳號.github.io/倉庫名稱/LOVEFIX-admin.html
用網址（https://）打開後，下面照樣填帳號／倉庫／權杖，「連接並初始化」就會正常運作（此時只是驗證，不用再重新建立）。

① GitHub 帳號名稱

② 倉庫（Repository）名稱

③ Personal Access Token

到 github.com → 右上角頭像 → Settings → 左側最下方 Developer settings → Personal access tokens → Fine-grained tokens → Generate new token。
Repository access 選「Only select repositories」（如果倉庫還沒建立，先選「All repositories」，之後在倉庫設定裡再收緊也可以）。
Permissions 底下找到 Contents 設為 Read and write，Administration 設為 Read and write（用來自動建立倉庫、開啟 Pages）。
按 Generate token，複製產生的字串貼在上面。這組權杖只會儲存在你目前這台電腦的瀏覽器本機，不會傳給任何第三方，請勿分享給別人。
